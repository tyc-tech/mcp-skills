# 合同主体核验

## 触发条件

当用户意图涉及以下场景时触发本 Skill：

- 签订合同前需要核验对方公司资质和信用状况
- 检查合同主体是否存在失信、被执行、经营异常等风险
- 进行合同签约前的尽职调查（主体维度）
- 用户提到"合同核验"、"签约前审查"、"合同主体风险"等关键词

## 输入要求

| 参数 | 是否必填 | 说明 |
|------|---------|------|
| keyword | 必填 | 公司名称、公司ID、注册号或统一社会信用代码 |
| code | 可选 | 注册号/组织机构代码/统一社会信用代码（用于三要素核验） |
| name | 可选 | 公司名称或机构名称（用于三要素核验） |
| legalPersonName | 可选 | 法人姓名（用于三要素核验） |

> 当 code、name、legalPersonName 三个参数均提供时，执行 Step 2 三要素验证；否则跳过该步骤。

## 执行流程

### Step 1: 查验工商登记状态

- 调用 `t1_ic_baseinfoV3` (参数: keyword) -- 获取企业基本信息，确认公司名称、经营状态（存续/注销/吊销）、成立日期、注册资本、法定代表人等核心信息

### Step 2: 三要素验证（条件执行）

- 调用 `t1_ic_verify` (参数: code, name, legalPersonName) -- 验证企业名称、法人、统一社会信用代码三者是否匹配一致；仅当三个参数均已提供时执行

### Step 3: 司法执行风险

- 调用 `t1_jr_zhixinginfo` (参数: keyword) -- 查询被执行人信息，关注执行金额和案件数量
- 调用 `t1_jr_dishonest` (参数: keyword) -- 查询失信被执行人信息，属于高风险信号

### Step 4: 经营异常与违法记录

- 调用 `t1_mr_abnormal` (参数: keyword) -- 查询经营异常名录，关注列入原因和日期
- 调用 `t1_mr_illegalinfo` (参数: keyword) -- 查询严重违法信息，属于高风险信号

### Step 5: 清算与注销状态

- 调用 `t1_mr_liquidating` (参数: keyword) -- 查询清算信息，判断企业是否正在清算
- 调用 `t1_mr_briefCancel` (参数: keyword) -- 查询简易注销信息，判断企业是否正在走注销流程

### Step 6: 综合风险评估

- 调用 `t1_risk_riskInfo` (参数: keyword) -- 获取天眼查综合风险评分，汇总各维度风险指标

- 法定代表人：
- 经营状态：
- 成立日期：
- 注册资本：

## 二、三要素验证结果（如执行）

- 被执行记录：X 条，合计金额 X 元

- 经营异常：X 条（列入/已移出）

## 错误处理

- 任一 Tool 调用返回 `error_code != 0`，记录错误信息，在报告中标注"该项查询失败，建议人工复核"
- 如 Step 1 企业基本信息查询失败（企业不存在），直接终止流程并提示"未查询到该企业信息，请核实公司名称或代码"
- 如遇"服务暂时不可用"（熔断），在报告中标注"该项服务暂不可用，建议稍后重试"
- 部分步骤失败不影响其他步骤执行，最终报告中标注缺失项

## 示例

**用户输入：**
> 帮我核验合同对方"北京某某科技有限公司"的主体资质

**执行过程：**
1. 调用 `t1_ic_baseinfoV3`(keyword="北京某某科技有限公司") -> 获取基本信息，经营状态为"存续"
2. 跳过三要素验证（未提供 code/name/legalPersonName）
3. 调用 `t1_jr_zhixinginfo` -> 发现 2 条被执行记录；调用 `t1_jr_dishonest` -> 无失信记录
4. 调用 `t1_mr_abnormal` -> 1 条已移出；调用 `t1_mr_illegalinfo` -> 无记录
5. 调用 `t1_mr_liquidating` -> 无清算；调用 `t1_mr_briefCancel` -> 无注销
6. 调用 `t1_risk_riskInfo` -> 风险评分中等

**输出：** 生成合同主体核验报告，风险等级为"中风险"，提示关注被执行记录。
