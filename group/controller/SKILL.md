# 实控人图谱

## 触发条件

当用户意图涉及以下场景时触发本 Skill：

- 识别企业背后的实际控制人
- 分析实控人的控制版图和关联企业
- 评估实控人个人风险对企业的影响
- 用户提到"实控人"、"实际控制人"、"最终受益人"、"股权穿透"、"谁控制这家公司"等关键词

## 输入要求

| 参数 | 是否必填 | 说明 |
|------|---------|------|
| keyword | 必填 | 公司名称、公司ID、注册号或统一社会信用代码 |

> Step 5 和 Step 6 中的 name 和 humanName 参数取自前序步骤的返回结果。name 为目标企业名称，humanName 为识别出的实控人姓名。

## 执行流程

### Step 1: 基本信息

- 调用 `t1_ic_baseinfoV3` (参数: keyword) -- 获取企业基本信息，确认企业名称、法定代表人等基础信息

### Step 2: 疑似实际控制人

- 调用 `t1_ic_actualControl` (参数: keyword) -- 获取股权向上穿透后识别的疑似实际控制人，包括控制人姓名、层级关系等

### Step 3: 最终受益人

- 调用 `t1_ic_humanholding` (参数: keyword) -- 获取股权向上穿透后识别的企业最终受益人，包括股东姓名、持股比例、层级关系等

### Step 4: 股权控制结构

- 调用 `t1_services_v4_open_equityRatio` (参数: keyword) -- 获取企业股权控制结构及疑似实际控制人，展示完整股权穿透链路

### Step 5: 实控人控股企业

- 调用 `t1_human_companyholding` (参数: name=企业名称, humanName=实控人姓名) -- 获取实控人名下控股的所有企业信息，包括企业名称、投资比例、经营状态等

### Step 6: 实控人个人风险

- 调用 `t1_services_v4_open_humanRiskInfo` (参数: name=企业名称, humanName=实控人姓名) -- 获取实控人相关的天眼风险列表，包括个人周边风险和预警信息

## 错误处理

- 任一 Tool 调用返回 `error_code != 0`，记录错误信息，在报告中标注"该项查询失败，建议人工复核"
- 如 Step 1 企业基本信息查询失败（企业不存在），直接终止流程并提示"未查询到该企业信息，请核实公司名称或代码"
- 如 Step 2 未识别到实际控制人，标注"该企业暂未识别到疑似实际控制人"，Step 5/6 跳过
- 如遇"服务暂时不可用"（熔断），在报告中标注"该项服务暂不可用，建议稍后重试"
- 部分步骤失败不影响其他步骤执行，最终报告中标注缺失项

## 示例

**用户输入：**
> 帮我分析"某某科技有限公司"的实际控制人是谁，以及他还控制了哪些企业

**执行过程：**
1. 调用 `t1_ic_baseinfoV3`(keyword="某某科技有限公司") -> 法定代表人为李某
2. 调用 `t1_ic_actualControl` -> 疑似实控人为张某（通过 3 层持股控制）
3. 调用 `t1_ic_humanholding` -> 最终受益人张某（持股比例 65%）
4. 调用 `t1_services_v4_open_equityRatio` -> 股权控制链路：张某 -> A公司(80%) -> B公司(70%) -> 某某科技(55%)
5. 调用 `t1_human_companyholding`(name="某某科技有限公司", humanName="张某") -> 张某控股 12 家企业
6. 调用 `t1_services_v4_open_humanRiskInfo`(name="某某科技有限公司", humanName="张某") -> 张某个人风险低

**输出：** 生成实控人图谱报告，确认实控人为张某，通过 3 层股权穿透控制目标企业，名下共控股 12 家企业，个人风险低。
